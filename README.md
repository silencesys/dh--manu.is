# Contributing to Manu

Thank you for taking the time to open an issue. Manu is a solo project. I maintain it alongside research and other work, so I may not always respond immediately, but I do read everything.

---

## Before opening an issue

A quick check first:

- **Search existing issues** — someone may have already reported the same bug or asked the same question.
- **Make sure you are using the latest version** — some bugs get fixed quietly between releases.

---

## Opening an issue

Use one of the templates below. They ask for the right information upfront, which makes it much easier for me to help:

| Template | Use it when... |
|---|---|
| [Bug report](.github/ISSUE_TEMPLATE/bug_report.md) | Something isn't working the way it should |
| [Feature request](.github/ISSUE_TEMPLATE/feature_request.md) | You have an idea for something new or improvements |
| [Q&A Discussion](../../discussions/new?category=q-a) | You are not sure how something works or need a hand getting set up |

---

## A few things that help

- **Be specific.** "It doesn't work" is hard to act on. "The annotation panel doesn't open on Firefox 124 when the layer has more than 50 annotations" is something I can reproduce and fix.
- **Include your environment.** Browser, OS, and deployment method all matter.
- **Share the manifest URL if relevant.** Many issues are IIIF-source-specific. If you can point me at the manifest, I can test directly.
- **Paste error messages as text, not screenshots.** It makes them searchable.

---

## What to expect

I triage issues as I can. Bugs that affect core functionality get priority. Feature requests go into a backlog, and I will comment if I plan to pick something up. If an issue goes quiet for a long time without enough information to act on, I may close it. Feel free to reopen or comment if you have more to add.
